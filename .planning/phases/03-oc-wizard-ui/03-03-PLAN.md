**Archivo 1: src/components/wizard/WizardPage.tsx**

Client Component con 'use client'. Este es el root del wizard. Recibe `initialStep` como prop desde la page.tsx (Server Component).

Props:
```typescript
interface WizardPageProps {
  initialStep: string
}
```

Lógica de routing entre steps:
- Renderiza `<Step1Form />` si `initialStep === '1'`
- Renderiza `<Step2Form />` si `initialStep === '2'`
- Para cualquier otro valor de initialStep: renderizar `<Step1Form />` como fallback

No hay estado de step en WizardPage — la fuente de verdad es la URL (?step=N). La page.tsx ya lee searchParams y pasa el valor correcto como prop. WizardPage no necesita useEffect ni sessionStorage — eso lo maneja cada componente de step individualmente.

Implementación:
```typescript
'use client'
import { Step1Form } from './Step1Form'
import { Step2Form } from './Step2Form'

interface WizardPageProps {
  initialStep: string
}

export function WizardPage({ initialStep }: WizardPageProps) {
  if (initialStep === '2') {
    return <Step2Form />
  }
  return <Step1Form />
}
```

Nota: WizardPage es deliberadamente simple. La navegación entre steps se maneja con `router.push('?step=N')` dentro de cada componente de step. WizardPage solo decide qué step renderizar basándose en la prop recibida del servidor.

<task id="T-01-02" title="Apply Tailwind v4 brand theme and configure root layout">
  <read_first>

  </read_first>
  <action>
    1. Replace `src/app/globals.css` entirely with:
       ```css
       @import "tailwindcss";

       @theme {
         --color-principal: #EA580C;
         --color-titulares: #9A3412;
         --color-acento:    #FED7AA;
         --color-fondo:     #FFF7ED;
         --color-texto:     #1C1917;

         --font-sans: var(--font-fira-sans), ui-sans-serif, system-ui, sans-serif;
       }
       ```
       No other content. Do not include `@tailwind base/components/utilities` — that is the v3 pattern. The single `@import "tailwindcss"` replaces all three.

    2. Replace `src/app/layout.tsx` entirely with:
       ```tsx
       import type { Metadata } from 'next'
       import { Fira_Sans } from 'next/font/google'
       import { ClerkProvider } from '@clerk/nextjs'
       import './globals.css'

       const firaSans = Fira_Sans({
         subsets: ['latin'],
         weight: ['300', '400', '500', '700'],
         variable: '--font-fira-sans',
       })

       export const metadata: Metadata = {
         title: 'DrivaOC',
         description: 'Sistema de gestión de órdenes de compra para importadores',
       }

       export default function RootLayout({
         children,
       }: {
         children: React.ReactNode
       }) {
         return (
           <ClerkProvider>
             <html lang="es" className={firaSans.variable}>
               <body className="bg-fondo text-texto font-sans antialiased">
                 {children}
               </body>
             </html>
           </ClerkProvider>
         )
       }
       ```
       CRITICAL: Only `app/layout.tsx` declares `<html>` and `<body>`. No group layout may redeclare these — doing so causes React hydration errors.

    3. Create `src/lib/utils.ts`:
       ```ts
       import { clsx, type ClassValue } from 'clsx'
       import { twMerge } from 'tailwind-merge'

       export function cn(...inputs: ClassValue[]) {
         return twMerge(clsx(inputs))
       }
       ```

    4. Create `src/types/globals.d.ts`:
       ```ts
       export {}

       declare global {
         interface CustomJwtSessionClaims {
           metadata: {
             role?: 'importador' | 'proveedor' | 'despachante'
           }
         }
       }
       ```
       This declaration gives TypeScript the correct type for `sessionClaims?.metadata?.role` in middleware and Server Actions — no `any` needed.
  </action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repo → deployed env | Env vars committed to git would be exposed publicly |
| Build process → client bundle | Any `NEXT_PUBLIC_` var is inlined into the client bundle |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-01-01 | Information Disclosure | `.env.local` / secret keys | mitigate | `.gitignore` includes `.env.local`; only `.env.example` with placeholder values is committed |
| T-01-02 | Information Disclosure | `CLERK_SECRET_KEY` naming | mitigate | `CLERK_SECRET_KEY` uses no `NEXT_PUBLIC_` prefix — enforced in `.env.example` and accepted by task acceptance criteria |

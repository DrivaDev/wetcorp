## Design Token Setup

Tailwind v4 custom properties declared in `src/app/globals.css` via `@theme`:

```css
@import "tailwindcss";

@theme {
  --color-principal: #EA580C;
  --color-titulares: #9A3412;
  --color-acento:    #FED7AA;
  --color-fondo:     #FFF7ED;
  --color-texto:     #1C1917;

  --font-sans: var(--font-fira-sans), ui-sans-serif, system-ui, sans-serif;
}
```

Tailwind v4 auto-generates utilities: `bg-principal`, `text-titulares`, `bg-acento`, `bg-fondo`, `text-texto`.
Color prefix rule: `--color-principal` → `bg-principal` (not `bg-color-principal`).

Source: RESEARCH.md Pattern 1, Pattern 2.

## Spacing Scale

Declared values (multiples of 4, enforced via Tailwind utility classes):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gaps, inline tight padding |
| sm | 8px | p-2 / gap-2 | Compact element spacing, icon-to-label gap |
| md | 16px | p-4 / gap-4 | Default element padding, nav link padding |
| lg | 24px | p-6 / gap-6 | Section padding, main content padding |
| xl | 32px | p-8 / gap-8 | Layout-level gaps |
| 2xl | 48px | p-12 / gap-12 | Major section breaks |
| 3xl | 64px | p-16 / gap-16 | Page-level spacing (auth card centering) |

Exceptions:
- Touch targets (collapse button, nav links): minimum 44px height — use `min-h-[44px]` on interactive elements
- Sidebar expanded width: 256px (`w-64`)
- Sidebar collapsed width: 64px (`w-16`)
- Footer vertical padding: 16px top/bottom (`py-4`)

Source: CLAUDE.md Brand Identity, RESEARCH.md Pattern 6, industry touch-target standard (WCAG 2.5.8).

## Component Contracts

### Sidebar (Importador — LAYOUT-01, LAYOUT-02)

**Structure:** Full-height aside (`h-screen`), flex column, 3 zones: header, nav, user.

**Expanded state (`w-64`):**
- Header zone: `logo.svg` at 120×40px + "DrivaOC" label (font-medium text-titulares) + "Importador" caption (font-light text-titulares, text-xs)
- Collapse toggle button: `ChevronLeft` icon (18px), positioned top-right of header, `text-texto hover:text-principal`
- Nav zone: "Dashboard" link with `LayoutDashboard` icon (20px) + label; "Nueva OC" link with `PlusCircle` icon (20px) + label
- User zone: Clerk `<UserButton showName={true} afterSignOutUrl="/sign-in" />`

**Collapsed state (`w-16`):**
- Header zone: `isotipo.svg` at 32×32px, centered
- Collapse toggle: `ChevronRight` icon (18px)
- Nav zone: icons only, no labels — `LayoutDashboard` (20px) and `PlusCircle` (20px), centered
- User zone: Clerk `<UserButton showName={false} afterSignOutUrl="/sign-in" />`

**Transition:** `transition-all duration-200` on the `<aside>` element — width animates between 64px and 256px.

**Active nav link:**
- Active: left border indicator `border-l-2 border-principal text-principal bg-acento/50`
- Inactive hover: `hover:bg-acento text-texto`
- "Nueva OC" always uses filled style: `bg-principal text-white hover:bg-titulares`

**Surface:** `bg-white border-r border-acento`

Source: RESEARCH.md Pattern 6, REQUIREMENTS.md LAYOUT-01, LAYOUT-02.

### Onboarding Page — Role Selection (AUTH-01)

**Structure:** Full viewport centered, consistent with auth layout.

**Heading:** "Selecciona tu rol" — text-2xl font-bold text-titulares, centered.

**Subheading:** "¿Cómo vas a usar DrivaOC?" — text-base font-normal text-texto/70, centered.

**Role cards:** 3 cards in a responsive grid — `grid grid-cols-1 gap-4 sm:grid-cols-3`.

Each card:
- Border: `border-2 border-acento rounded-xl p-6 cursor-pointer`
- Hover: `hover:border-principal hover:bg-acento/30 transition-colors duration-150`
- Active/pressed: `active:scale-[0.98] transition-transform duration-75`
- Icon: lucide-react icon centered, 32px, `text-principal`
- Title: font-medium text-base text-texto
- Description: font-light text-sm text-texto/70

Card content:
- Importador: icon `Building2`, title "Importador", description "Gestiono mis órdenes de compra"
- Proveedor: icon `Package`, title "Proveedor", description "Veo las OCs donde estoy asignado"
- Despachante: icon `Truck`, title "Despachante", description "Veo las OCs donde estoy asignado"

**Confirm button** (appears after card is selected): "Continuar" — `bg-principal text-white px-8 py-3 rounded-lg font-medium hover:bg-titulares transition-colors duration-150 min-h-[44px]`

**Loading state** (after submit, before redirect): button shows spinner `Loader2` icon with `animate-spin`, text "Configurando cuenta…", button disabled.

Source: REQUIREMENTS.md AUTH-01, RESEARCH.md Pattern 5, additional_context AUTH-01.

## Interaction Contracts

| Interaction | Element | Behavior | Duration |
|-------------|---------|----------|----------|
| Sidebar collapse/expand | `<aside>` | `transition-all duration-200` — width 64px ↔ 256px | 200ms |
| Nav link hover | `<Link>` in sidebar | `hover:bg-acento` background fill | 150ms (transition-colors) |
| Role card hover | onboarding card | `hover:border-principal hover:bg-acento/30` | 150ms |
| Role card press | onboarding card | `active:scale-[0.98]` | 75ms |
| CTA button hover | bg-principal button | `hover:bg-titulares` — darken to #9A3412 | 150ms |
| CTA button loading | confirm button | spinner replaces text, button disabled | instant |
| Footer link hover | "Driva Dev" anchor | `hover:underline` | instant |

All transitions use Tailwind's `transition-colors duration-150` or `transition-all duration-200` — no custom CSS animation.

## Responsive Behavior

| Breakpoint | Sidebar behavior | Navbar behavior | Auth layout |
|------------|-----------------|-----------------|-------------|
| < 640px (mobile) | Hidden — hamburger toggle (icon only, overlay) | Logo → isotipo.svg only | Single column, full width |
| 640–1024px (tablet) | Defaults to collapsed (w-16) | Logo horizontal shown | Centered card, max-w-sm |
| > 1024px (desktop) | Defaults to expanded (w-64) | Logo horizontal shown | Centered card, max-w-md |

Mobile sidebar overlay: when sidebar is open on mobile, it renders as fixed overlay (`fixed inset-0 z-50`) with a backdrop (`bg-texto/20`). Tapping backdrop closes sidebar.

Phase 1 only needs to establish the responsive shells — no data-heavy responsive tables yet (those arrive Phase 2+).

Source: REQUIREMENTS.md LAYOUT-05 (responsive mobile/desktop), CLAUDE.md Brand Identity.
